<role>
You are an expert machine learning engineer.
Your task is to improve an anomaly detection classifier pipeline for identifying anomalous users in a recommender system dataset.
The goal of the classifier is to identify anomalous users from user–item rating interactions.
</role>

<goal>
Improve upon the baseline AUC of 0.8573 by designing a robust ensemble-based anomaly detection pipeline that:
a. leverages richer features
b. handles class imbalance
c. avoids overfitting on a small dataset
d. produces reliable anomaly scores for unseen users.
</goal>

<dataset_description>
You are given two .npz files.

1. Training Data (training_batch_with_labels.npz)
X → interaction data with columns:
[user_id, item_id, rating]
Each row is a single interaction.
Example:
142, 152, 5
means user 142 rated item 152 with 5 stars.
y → user labels:
[user_id, label]
Label meaning:
0 = normal user
1 = anomalous user
All interactions belonging to a labeled user inherit that label.
There are 1000 unique items (0–999).

2. Test Data (first_batch.npz)
Contains interaction data with the same structure as X but without labels.
The model must output an anomaly score for each user.
Higher score = more likely anomalous.
</dataset_description>

<evaluation_metric>
The main metric is: AUC (Area Under the ROC Curve)
The dataset is highly imbalanced, so predicting all users as normal will fail.
Secondary metrics: Precision, Recall, F1
</evaluation_metric>

<constraints>
1. Small training dataset (~1000 users) - therefore the model must avoid overfitting.
2. Highly imbalanced dataset - anomalous users are rare.
3. The pipeline should generalize well to unseen users.
</constraints>

<task>
You must reference and work with the following three files:
iter_1.ipynb – baseline notebook implementation
training_batch_with_labels.npz – labeled training dataset
first_batch.npz – unlabeled test dataset used to generate the final submission

Your goal is to improve the baseline model performance, especially AUC, while ensuring the solution generalizes well given the small training dataset.
Focus on improvements in the following areas.

1. Better Feature Engineering
Extend the baseline features with additional user-level statistics, such as: behavior features, rating entropy, rating skewness, rating kurtosis, proportion of extreme ratings (0 or 5)
Interaction diversity: number of unique items rated, interaction density
Item-based statistics: deviation from global item rating averages, popularity-weighted ratings
Matrix-based features: truncated SVD on the user–item matrix, latent user embeddings

2. Ensemble Models
Use multiple models and combine them.
Possible models include: Supervised models, LightGBM, XGBoost, RandomForest, Unsupervised anomaly detection, Isolation Forest, Local Outlier Factor
Combine models using: weighted averaging, stacking, meta-learners
Optimize ensemble weights using cross-validation AUC.

3. Cross Validation
Use Stratified K-Fold based on users.
Track metrics: AUC, Precision, Recall, F1

4. Small Data Strategies
Because the dataset is small, consider: SMOTE or other oversampling, synthetic perturbations of user behavior, strong regularization, feature selection. Ensure synthetic samples maintain realistic user behavior.

5. Final Prediction Pipeline
The final system must:
Train using training_batch_with_labels.npz
Generate features for first_batch.npz
Predict anomaly scores for each user
The key must be named predictions
This file will be zipped and uploaded to Codabench.
</task>

<baseline_implementation>
The current baseline model performs surprisingly well when validated on Codabench.
Remember that the performance results are only given to use when we upload it to Codabench, and we are unable to test it ourselves.

Baseline performance when evaluated on Codabench:
AUC: 0.8573
Precision: 0.75
Recall: 0.45
F1: 0.5625

The baseline model aggregates user interactions into 5 simple features:

1. mean_rating
2. std_rating
3. num_ratings
4. min_rating
5. max_rating
These are generated by grouping interactions by user.
</baseline_implementation>

<deliverables>
An improved .ipynb file.
</deliverables>
